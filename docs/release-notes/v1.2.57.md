# Storage Redirect X v1.2.57

## 模块更新

### 增加了什么功能

- 完善文件监视能力：支持按应用、操作和路径规则筛选记录，显示重定向后的实际路径，并改进过滤规则的选择、排序和管理体验。
- 文件监视现在覆盖普通文件写入、已有文件覆盖、延迟关闭写入和系统媒体代写等更多操作链路。

### 修复了什么问题

- 修复普通应用重定向、路径映射、通配放行、只读目录及只读排除规则组合使用时的挂载和写入权限问题，并确保普通应用只使用 companion mount、mount namespace 或 scoped FUSE 路径。
- 修复 MediaStore 和系统媒体代写在相对路径、映射路径、只读目录及多应用共享系统写入进程下的归因与隔离问题，避免一个应用的规则影响其它应用，并保持只读目录中的已有媒体可查询。
- 修复配置热更新、应用启动和挂载变化期间重定向或文件监视可能延迟生效、短暂失效或漏记事件的问题。
- 修复模块更新等待重启时状态识别不准确的问题。

### 注意事项

- 更新模块后需要重启设备，使 Zygisk、系统媒体进程和挂载状态完整加载新版本。
- 已有配置可继续使用；涉及正在运行应用的挂载或媒体写入规则时，建议在升级重启后确认目标应用的实际读写落点。

## App 更新

### 增加了什么功能

- 新增检查更新入口和启动时自动检查，可按已安装模块版本判断更新，并从静态更新清单读取正式版与测试版信息。
- 更新弹窗现在按模块、App 和其它更新分类展示版本说明。
- App 与 WebUI 统一主题设置和动态取色体验，并完善文件监视过滤、操作显示和设置交互。

### 新功能怎么使用

- 在 App 的更新页面可手动检查版本；启动自动检查可在设置中调整，发现新版本后可直接查看分类更新说明和下载入口。
- 在文件监视页面使用应用、操作和路径过滤条件缩小记录范围，过滤规则列表会按名称排序以便查找。

### 修复了什么问题

- 修复更新检查版本来源、版本文案、仓库地址和模块版本尚未读取完成时的自动检查问题。
- 修复日志包导出进度可能倒退、窄屏状态显示和部分弹窗背景或选择状态不一致的问题。

## 其它更新

### 端到端测试流

- 端到端测试流已内置到仓库，覆盖重定向、路径映射、只读规则、MediaStore、文件监视、配置热更新和模块启动状态等 scenario 1-29。
- PR、CI Build 和正式 Release 会使用一次构建的 x86_64 模块与测试 APK，在 Android 13、14、15、16 模拟器上分别运行完整场景；只有全部版本通过 `Test-flow required gate` 后才允许发布资产或更新清单。
